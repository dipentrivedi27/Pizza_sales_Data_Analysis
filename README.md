# 🍕 Pizza Sales Data Analysis (SQL Project)

## 📌 Project Overview
This project focuses on analyzing a pizza sales dataset using SQL to extract meaningful business insights. The goal is to understand sales performance, customer behavior, and product trends to support data-driven decision-making.

---

## 🎯 Objectives
- Analyze overall sales performance  
- Identify top-selling pizzas and categories  
- Understand customer ordering patterns  
- Discover peak order times  
- Generate insights for business growth  

---

## 🗂️ Dataset Description
The dataset contains information related to pizza orders, including:
- Order details  
- Pizza types and categories  
- Quantity and pricing  
- Date and time of orders  

---

## 🛠️ Tools & Technologies
- **SQL (MySQL)**
- Concepts Used:
  - SELECT
  - JOIN
  - GROUP BY
  - ORDER BY
  - Aggregate Functions (SUM, COUNT, AVG)

---

## 📊 Analysis Tasks

### 🔹 Basic Analysis
1. Retrieve the total number of orders placed  
2. Calculate total revenue generated from pizza sales  
3. Identify the highest-priced pizza  
4. Find the most common pizza size ordered  
5. List top 5 most ordered pizza types with quantities  

---

### 🔹 Intermediate Analysis
6. Find total quantity of each pizza category ordered (using joins)  
7. Analyze order distribution by hour of the day  
8. Category-wise distribution of pizzas  
9. Average number of pizzas ordered per day  
10. Top 3 pizza types based on revenue  

---

### 🔹 Advanced Analysis
11. Percentage contribution of each pizza type to total revenue  
12. Cumulative revenue over time  
13. Top 3 pizza types by revenue for each category  

---

## 📈 Key Insights
- Peak order hours help optimize staffing  
- Certain pizza categories dominate sales  
- A few pizza types contribute most of the revenue (Pareto Principle)  
- Revenue trends show business growth patterns over time

3. Analyze outputs and interpret insights
